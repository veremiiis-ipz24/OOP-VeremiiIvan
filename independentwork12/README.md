# Самостійна робота №12

## Тема: PLINQ: дослідження продуктивності та безпеки

## Мета

Дослідити переваги та недоліки PLINQ у порівнянні зі звичайним LINQ, виміряти продуктивність на різних обсягах даних та проаналізувати проблеми потокобезпечності.

---

## Опис виконання роботи

У межах роботи було створено консольний застосунок **IndependentWork12**, який:

* генерує великі масиви даних (1M, 5M, 10M елементів)
* виконує обчислювально складну операцію (перевірка на просте число + обчислення кореня)
* порівнює час виконання LINQ та PLINQ
* демонструє проблему побічних ефектів у паралельних обчисленнях

---

## Дослідження продуктивності

### Вхідні дані

Використано колекції розміром:

* 1 000 000 елементів
* 5 000 000 елементів
* 10 000 000 елементів

Числа генеруються випадковим чином.

### Обчислювальна операція

Для кожного елемента:

* перевірка числа на простоту
* обчислення квадратного кореня

### Методика вимірювання

Час виконання вимірюється за допомогою `Stopwatch` окремо для:

* LINQ
* PLINQ

### Результати (типові)

* при малих обсягах різниця незначна
* при великих обсягах PLINQ працює швидше
* інколи PLINQ може бути повільнішим через накладні витрати

---

## Аналіз результатів

PLINQ показує кращу продуктивність у випадках:

* великих обсягів даних
* обчислювально складних операцій
* відсутності блокувань та залежностей між потоками

PLINQ може бути неефективним, якщо:

* обсяг даних малий
* операції прості
* є витрати на синхронізацію

---

## Дослідження побічних ефектів

### Проблема

Було створено сценарій, у якому змінюється спільна змінна всередині PLINQ:

```csharp
int sum = 0;

data.AsParallel().ForAll(x =>
{
    sum += x;
});
```

### Результат

Отримується некоректне значення змінної `sum`, оскільки:

* кілька потоків одночасно змінюють одну змінну
* відсутня синхронізація

---

## Виправлення проблеми

Було використано механізм `lock`:

```csharp
int sum = 0;
object locker = new object();

data.AsParallel().ForAll(x =>
{
    lock (locker)
    {
        sum += x;
    }
});
```

### Результат

* результат стає коректним
* забезпечується потокобезпечність

---

## Висновки

У ході роботи встановлено:

1. PLINQ дозволяє ефективно використовувати багатоядерність процесора.
2. Найбільший ефект досягається при великих обсягах даних та складних обчисленнях.
3. Для простих задач використання PLINQ може бути недоцільним.
4. Побічні ефекти у паралельному коді можуть призводити до некоректних результатів.
5. Для безпечної роботи необхідно використовувати механізми синхронізації або уникати спільного стану.

PLINQ є потужним інструментом, але його використання потребує розуміння принципів багатопоточності.
