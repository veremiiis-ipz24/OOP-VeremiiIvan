# Лабораторна робота №31

## Тема

Тестування з використанням Moq (мокінг залежностей)

## Мета

Навчитися створювати mock-об'єкти за допомогою бібліотеки Moq для
тестування класів із залежностями, використовувати Setup та Verify.

## Виконання роботи

У роботі реалізовано сервіс **UserService**, який має дві залежності: -
IUserRepository --- робота з користувачами - IPasswordHasher ---
хешування пароля

Залежності передаються через конструктор (Dependency Injection).

## Опис реалізації

### Інтерфейси

-   IUserRepository:
    -   AddUser(...)
    -   UserExists(...)
-   IPasswordHasher:
    -   Hash(...)

### Сервіс

UserService реалізує: - Register(...) --- реєстрація користувача -
UserExists(...) --- перевірка існування - HashPassword(...) ---
хешування

## Тестування

Для тестування використано: - xUnit - Moq

### Використання Moq

Setup --- налаштування поведінки mock:

    repoMock.Setup(r => r.UserExists("john")).Returns(false);

Verify --- перевірка викликів:

    repoMock.Verify(r => r.AddUser("john", "HASH"), Times.Once);

## Реалізовані тести

У проєкті реалізовано 8 тестів: 1. Успішна реєстрація 2. Відмова при
існуючому користувачі 3. Виклик хешування пароля 4. Виклик додавання
користувача 5. Перевірка UserExists (true) 6. Перевірка UserExists
(false) 7. Перевірка HashPassword 8. Перевірка виклику Hash

## Висновок

У ході роботи було отримано практичні навички: - використання моків -
ізоляція залежностей - перевірка взаємодії між компонентами

Moq значно спрощує тестування, дозволяючи контролювати поведінку
залежностей і перевіряти виклики методів.
