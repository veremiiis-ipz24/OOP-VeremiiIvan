# Самостійна робота №11

## Тема: Кейси Polly / Retry

## Мета

Дослідити та проаналізувати застосування бібліотеки Polly для реалізації політик відмовостійкості (Retry, Circuit Breaker, Timeout) у .NET-застосунках.

---

## Опис виконання роботи

У межах роботи було створено консольний застосунок **IndependentWork11**, у якому реалізовано декілька сценаріїв використання бібліотеки Polly.

---

## Сценарій 1. Виклик зовнішнього API (Retry)

### Проблема

Зовнішній API може тимчасово повертати помилки (мережеві збої, перевантаження).

### Обґрунтування вибору

Використано політику Retry, оскільки помилки є тимчасовими і запит можна повторити.

### Реалізація

Застосовано `WaitAndRetry` з експоненційною затримкою (2, 4, 8 секунд).
Перші два виклики штучно завершуються помилкою.

### Очікувана поведінка

* перші виклики завершуються помилкою
* Polly виконує повторні спроби
* наступний виклик проходить успішно

---

## Сценарій 2. Доступ до бази даних (Circuit Breaker)

### Проблема

Постійні помилки підключення до бази можуть перевантажити систему.

### Обґрунтування вибору

Використано Circuit Breaker для тимчасового блокування викликів після серії помилок.

### Реалізація

Після 2 помилок ланцюг розривається на 5 секунд.

### Очікувана поведінка

* перші виклики викликають помилки
* після досягнення порогу виклики блокуються
* після паузи робота відновлюється

---

## Сценарій 3. Довга операція (Timeout)

### Проблема

Операція може виконуватися занадто довго.

### Обґрунтування вибору

Використано Timeout для обмеження часу виконання.

### Реалізація

Операція штучно затримується на 4 секунди, тоді як timeout встановлено 2 секунди.

### Очікувана поведінка

* операція перевищує час
* Polly перериває виконання
* генерується виняток Timeout

---

## Логування

У програмі використано базове логування через `Console.WriteLine`, що дозволяє відстежувати:

* кількість спроб
* причини помилок
* спрацювання політик

---

## Висновки

У ході роботи встановлено:

1. Polly дозволяє ефективно реалізувати відмовостійкість у застосунках.
2. Retry доцільно використовувати для тимчасових збоїв.
3. Circuit Breaker запобігає перевантаженню системи при частих помилках.
4. Timeout дозволяє уникнути зависання операцій.
5. Комбінація політик забезпечує стабільну роботу застосунку в умовах нестабільного середовища.

Загалом, використання Polly є доцільним при роботі з зовнішніми сервісами та мережевими ресурсами.
