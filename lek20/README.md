# Lecture 20 – Adapter, Facade, Proxy (C#)

## Опис

У цьому проєкті реалізовано три структурні патерни проєктування:

- Adapter (Адаптер)
- Facade (Фасад)
- Proxy (Заступник)

Проєкт демонструє інтеграцію, спрощення підсистем та контроль доступу до об'єктів
у середовищі C# (.NET Console Application).

---

# 1. Adapter

## Призначення

Adapter перетворює інтерфейс одного класу на інший, який очікує клієнт.
Дозволяє використовувати існуючий клас із несумісним інтерфейсом.

## Реалізація

- IXmlProcessor — Target
- ThirdPartyJsonProcessor — Adaptee
- JsonToXmlAdapter — Adapter
- ClientApp — Client

## Логіка роботи

XML-інтерфейс адаптується до JSON-обробника.
Adapter виконує перетворення та делегує виклик Adaptee.

## Переваги

- Повторне використання існуючого коду
- Мінімальні зміни клієнтського коду

## Недоліки

- Додається додатковий рівень абстракції

---

# 2. Facade

## Призначення

Facade надає спрощений інтерфейс до складної підсистеми.

## Підсистема

- Amplifier
- DvdPlayer
- Projector

## Facade

- HomeTheaterFacade

## Логіка роботи

Метод WatchMovie() координує роботу всіх компонентів.
Метод EndMovie() коректно завершує роботу підсистеми.

## Переваги

- Зменшує складність клієнтського коду
- Інкапсулює підсистему

## Недоліки

- Може перетворитися на перевантажений клас

---

# 3. Proxy

## Призначення

Proxy контролює доступ до реального об'єкта.
У проєкті реалізовано Virtual Proxy (ледаче завантаження).

## Реалізація

- IImage — Subject
- RealImage — RealSubject
- ProxyImage — Proxy

## Логіка роботи

RealImage завантажується лише при першому виклику Display().
Повторні виклики не виконують повторне завантаження.

## Переваги

- Контрольований доступ
- Оптимізація ресурсів

## Недоліки

- Додатковий рівень абстракції
- Можлива затримка при першому зверненні
